01 Install

Requires Node 18+ and the Salesforce sf CLI, already logged in to your orgs (ferry reuses those credentials — it never asks you to log in again).

shell
# install once, globally
npm install -g sf-ferry

# …or run without installing
npx sf-ferry --help

02 Quick start

Point it at a source and a target. That's the whole setup.

shell
# browse uat, deploy to prod
ferry --source uat --target prod

# start from an existing package.xml or a metadata .zip
ferry --source uat --import manifest/package.xml

ferry opens the selector, connects to the source org under a splash, and lists its metadata types. Open a type to load its components, check what you want, press d to deploy or v to validate. Your selection and last target are remembered for next time.

You can skip the flags entirely. Run ferry on its own and it prompts you to pick a source org, then a target inside the UI with t.

03 The interface

04 Commands

The UI is the default command. The rest are for inspecting state and for automation.

CommandWhat it does
ferryThe interactive selector → validate / deploy. Aliases: go, ui, deploy.
ferry runNon-interactive deploy / validate for CI — selection from --import or --session, with --json output.
ferry logRecent deploy / validate history — what shipped where, when, and whether it passed.
ferry statusCached state: saved sessions, metadata cache, retrieve zips.
ferry cleanRemove cached state (--all also clears saved sessions).
ferry orgsList the orgs the sf CLI is authenticated to.

05 How it works

A deploy is really two hops — pull the components off the source, push them to the target. ferry does both over the sf CLI, in metadata format end to end.

BROWSE

Read live metadata

One describe call for the type list, then one list call per type you open — for owner, created & modified dates. Cached under ~/.ferry.

→
SELECT

Pick components

Check components across any number of types. The Selected pane is your running manifest.

→
RETRIEVE

Pull from source

ferry writes package.xml and retrieves exactly your selection from the source org as a single unpackaged.zip.

→
DEPLOY

Push to target

That zip is deployed (or validated) to the target with your chosen test level. Output streams live.

Cached retrieve, so retries are instant

After a retrieve, ferry fingerprints the selection. If a deploy fails and you retry with the same components, it reuses the downloaded zip instead of pulling again — you'll see reused the cached zip. Changed the selection, or changed the components in the source org? Pass --refetch to force a fresh pull.

Nothing is lost between runs

Your selection is checkpointed to history on every deploy — and on save & quit — so an abandoned or failed run comes back pre-checked via the s picker. ferry also remembers your last target org and last-open type per source, and restores them on the next launch.

06 Automate (CI)

ferry run is the headless path: no UI, a defined selection in, a machine-readable result out. Ideal for a pipeline that validates every package against production.

ci
# validate a manifest against prod, JSON result for the pipeline
ferry run --source uat --target prod \
  --import manifest/package.xml \
  --validate --test-level RunLocalTests --json

# → {"ok":true,"action":"validate","count":24,"code":0, …}

Swap --import for --session release-1.4 to replay a selection you named in the UI with S. Drop --validate to actually deploy. Exit code mirrors the deploy result.

07 Where state lives

Everything ferry remembers is under ~/.ferry (override with FERRY_HOME), keyed by org username so it follows you across folders. Inspect it with ferry status; the only thing written into your project is the package.xml.

PathHolds
cache/<org>/<Type>.jsonMetadata listings. Never expires — the UI shows the age; r re-pulls.
sessions/<org>.jsonYour last 20 selections, including named ones.
retrieve/<org>/unpackaged.zipThe retrieved metadata + a selection fingerprint for cache reuse.
prefs.jsonLast target org and last-open type, per source.
log.jsonDeploy / validate history — see ferry log.
config.jsonOptional defaults: apiVersion, defaultTestLevel.
Writes are atomic — a crash or ⌃C mid-save can never truncate a session or the log. Set NO_COLOR and ferry's text output drops ANSI for clean CI logs.
